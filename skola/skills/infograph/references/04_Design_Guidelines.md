# Design Guidelines

Rules for layout, color, typography, and iconography in technical infographics.

## Layout
* **Grid system**: Use consistent grid for alignment
* **One focal area**: Clear visual hierarchy with single entry point
* **Baseline alignment**: Text elements aligned to common baseline
* **Spacing**: 16–24px increments for consistent rhythm
* **Margins**: Adequate white space around all elements
* **Grouping**: Related elements close together, unrelated elements separated

## Color
* **1 accent color**: Primary brand or emphasis color
* **Semantic roles**:
  - Success: Green
  - Error/Warning: Red/Orange
  - Info: Blue
  - Neutral: Gray scale
* **Light/dark palettes**: Provide variants for different contexts
* **Avoid "rainbows"**: Too many colors create visual noise
* **Contrast**: Maintain WCAG AA ratios (4.5:1 normal, 3:1 large text)

## Typography
* **Display font**: Titles and headers (32px, 24px)
* **Sans serif**: Body text and labels (18px, 14px)
* **Monospace**: Code snippets and technical identifiers (14px, 12px)
* **Fixed scale**: 32/24/18/14/12 (avoid arbitrary sizes)
* **Line height**: 1.4–1.6 for readability
* **Weight hierarchy**: Bold for emphasis, regular for body

## Icons & Metaphors
* **Database**: Cylinder shape
* **Queue/Message**: Envelope icon
* **Nodes/Edges**: Circles connected by lines
* **Arrows for flow**: Indicate direction and data movement
* **Label protocols**: HTTP, gRPC, WebSocket clearly marked
* **Standard conventions**: Use recognized symbols (cloud, server, user)
* **Consistent style**: All icons from same family/style
