# Core Principles

## Clarity
* **One dominant message**: Single clear takeaway
* **Delete non-teaching elements**: Remove decorative elements that don't support learning
* **Data-ink mindset**: Maximize information density while maintaining readability

## Hierarchy
* **Structure**: Title → Key claim → Sections → Annotations
* **Visual priority**: Size, weight, and position over color
* **Intentional emphasis**: Guide eye to most important information first

## Flow
* **Reading patterns**: Z/F patterns for Western audiences
* **Numbered steps**: Sequential processes clearly labeled
* **Minimal line crossings**: Reduce visual complexity
* **Orthogonal connectors**: Use right angles for clarity and professionalism

## Cognitive Load
* **Chunk size**: 3–7 information chunks (Miller's Law)
* **Progressive disclosure**: Build complexity gradually
* **Pattern reuse**: Replace repeated text with icons and legends
* **Consistency**: Maintain visual language throughout

## Accessibility
* **Contrast**: WCAG AA minimum (4.5:1 for normal text, 3:1 for large text)
* **Color-independent cues**: Never rely on color alone to convey information
* **Alt text**: Include both structure description and key takeaway
* **Text size**: Minimum 14px for body text, 12px for annotations
