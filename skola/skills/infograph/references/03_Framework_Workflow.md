# Framework Workflow

Step-by-step process from concept to rendered infographic.

## 1. Define
* **Audience**: Who will view this? (Students, engineers, managers)
* **Goal**: What should they learn or do?
* **Single takeaway**: One sentence summary of the message

## 2. Classify
Choose the infographic type based on content:
* **Data**: Comparisons, metrics, statistics (use matrices, charts)
* **Process**: Sequential steps, algorithms, workflows (use flows, ladders)
* **Conceptual**: Relationships, patterns, architectures (use maps, diagrams)
* **Hybrid**: Combination of above (use modular composition)

## 3. Pick Template & Sketch Wireframe
* Select appropriate template from `05_Templates.md`
* Create numbered wireframe showing regions
* Blocks first, details later
* Label regions by type (title, content, code, diagram)

## 4. Fill Content
* **≤80 words per panel**: Keep text concise
* **Label edges**: Use verb + noun (e.g., "sends request", "returns data")
* **Code formatting**: Monospace font, syntax highlighting
* **Annotations**: Brief explanations for complex elements

## 5. Apply Style Tokens
* **Type scale**: Use fixed sizes (32/24/18/14/12)
* **Color**: 1 accent color + neutral scale
* **Stroke weight**: Consistent line weights (1px, 2px, 4px)
* **Spacing**: Grid-based (16–24px increments)

## 6. Validate
* **5-second test**: Can viewer identify main message quickly?
* **Teach-back test**: Can viewer explain concept after viewing?
* **Run checklist**: Verify all items in `08_Checklist.md`
* **Export variants**: Multiple formats and sizes for different platforms
